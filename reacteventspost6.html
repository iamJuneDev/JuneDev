React Events

How does react manage state changes?

We all know about Event handlers or event listeners, such as `onClick`, which are key to our function components.

The biggest difference between DOM element event handlers and React event-handlers are in syntax…

Remember that we use camelCase to name elements in react and we pass in a function as the event-handler instead of a string.

Other useful handlers in React are onSubmit, onChange, onKeyPress, onBlur.

In React we cannot use event handlers without adding proper functionality. To properly define our event we use constructor() method to have state and props hold values needed for the function component to render without errors.

In Redux, we take our programming skills further by implementing JavaScript and utilizing re-playable mutations, meaning our apps can be more easily maintained.

React core concept of encapsulation is a vital principal for application developments because it is the programming practice of breaking down components into smaller reusable components. By implementing JavaScript function components and JavaScript ES6 Classes, the work flow using React becomes more dynamic, cleaner, and easier to maintain and build with when having a large stream of data flow. The concept unidirectional data flow becomes essential when using state and props. By properly assigning events the functionality that is required in React, our code renders UI components that can be interchangeable and maintainable. Redux framework library is similar in it’s core concepts but offers solutions for state management.

Handling synthetic events with React elements is done by providing a listener when the element is initially rendered. Event handlers can also be methods in class, when defining a component using an ES6. It is recommended to bind constructor using the class fields syntax to avoid syntax errors in callback problems. When passing arguments to event handlers using Function.prototype.bind forwards argument when using e in the parameters.

My Coding Mentor Kevin puts it this way:

"I find that it helps to think about state and props in this manner: React components are essentially JavaScript functions that accept data and return a piece of the user interface. Props are like arguments and state is similar to the local scope variables used in a function! You don’t want to change the arguments that are given to a function but you can do whatever you want to the variables that you created within the function."

Now you’ve read briefly about React events and the state changes managed by this particular framework library. Learning the key elements and techniques that make React unique in composition will help you develop your app strategically and efficiently.
